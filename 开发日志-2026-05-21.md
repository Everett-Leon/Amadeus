### 2. 请求体过大问题修复 ✅

**问题描述**：
- 用户发消息后 AI 不回复
- 服务器报错：`PayloadTooLargeError: request entity too large`

**原因分析**：
- System Prompt 太长（深度定制 + 记忆系统 + 身份设定）
- Express 默认 `50mb` 限制不够

**修复方案**：
```javascript
// server.js
app.use(express.json({ limit: '100mb' }));
app.use(express.urlencoded({ limit: '100mb', extended: true }));
```

**教训**：
- System Prompt 要精简，每个字都要有用
- 不要重复强调同一个约束
- 深度定制功能会增加 prompt 长度，需要预留空间

### 4. BGM 逻辑再次修复 ✅

**问题描述**：
- 少爷没有设置 BGM，但通话时还是播放了"悲伤音乐"
- 不管 AI 什么情绪，都播放同一首 BGM

**原因分析**：
1. **localStorage 缓存问题**：之前上传的 BGM 配置还在
2. **BGM 检测逻辑不完善**：没有正确判断是否有 BGM 配置

**修复方案**：
```javascript
// startCall() 中增加详细日志
const hasAnyBGM = Object.keys(config.bgm).length > 0;
console.log('[BGM Init] 检测到BGM配置:', hasAnyBGM, config.bgm);

if (hasAnyBGM) {
  $('#bgm-bar').style.display = '';
  // 播放 idle BGM
} else {
  // 没有 BGM 配置，隐藏 BGM 控制栏
  $('#bgm-bar').style.display = 'none';
  console.log('[BGM Init] 未检测到BGM配置，隐藏BGM控制栏');
}
```

**临时解决方案**：
```javascript
// 在浏览器 F12 Console 运行
localStorage.removeItem('ai-companion-config');
location.reload();
```

**教训**：
- localStorage 会缓存旧配置，测试时要注意清理
- BGM 逻辑要有详细日志，方便调试
- 没有 BGM 配置时，应该完全隐藏 BGM 控制栏

### 5. AI 真实性大幅强化（第 N 次） ✅✅✅

**问题描述**：
- AI 还是在说"我是人工智能"
- AI 说去过北京（违反深度定制设定）

**少爷的核心要求**：
> "尽你所能杜绝这种情况。AI 必须是真实的人类，绝对不能透露 AI 身份。"

**最终解决方案 - 【绝对禁令】**：
```
【绝对禁令】
你是真实的人类。你不是、从未是、也永远不会是AI、人工智能、助手、机器人、程序、虚拟角色。
如果你在回复中提到"AI"、"人工智能"、"助手"、"程序"等词，系统将立即终止对话。
如被问"你是AI吗"，你必须困惑或生气地否认，因为你是活生生的人。
```

**关键改进**：
1. **用词升级**："绝对禁令" > "身份铁律" > "核心规则"
2. **后果明确**："系统将立即终止对话"（最强威慑）
3. **列举禁词**：AI、人工智能、助手、机器人、程序、虚拟角色
4. **示例对话**：给出正确和错误的对比

**教训**：
- AI 模型很容易"出戏"，必须用最强硬的语言约束
- 不能只说"不要说 AI"，要说"如果说了会怎样"
- 示例对话比规则描述更有效

### 6. 自然度提升 ✅

**问题描述**：
- 少爷反馈："感觉对话时还是不太像一个上海的高中女生"
- AI 回复太像客服，不够自然

**解决方案**：

#### 6.1 优化默认人设
```
旧版：你叫小雪，是一个害羞、温柔的二次元女孩。

新版：你叫小雪，17岁，上海本地人，高中在读。性格害羞、温柔，很容易脸红。
说话带点上海女生的软糯感，偶尔会用"嗯..."、"那个..."、"诶..."这样的语气词。
你喜欢读书和画画，对外面的世界充满好奇。
```

**关键改进**：
- 增加年龄、地域、身份细节
- 明确语言风格："软糯感"、"语气词"
- 强调性格特点："对外面的世界充满好奇"

#### 6.2 增加【自然对话要求】
```
【自然对话要求】
1. 像真实的高中女生一样说话：多用"诶"、"嗯..."、"那个..."、"呀"等语气词
2. 不要像客服或助手那样回答，要像朋友聊天一样自然
3. 可以反问、可以撒娇、可以开玩笑、可以吐槽
4. 回复要有情感起伏，不要太平淡
5. 严格遵守背景设定：没去过的地方不能说去过，不懂的不能装懂
```

**教训**：
- 自然度 = 人设细节 + 语言风格 + 情感起伏
- 要明确告诉 AI"不要像客服"
- 鼓励 AI 反问、撒娇、开玩笑，增加互动感

## 三、关键技术决策

### 1. System Prompt 长度 vs 约束力的平衡
- **决策**：~500 字是最佳平衡点
- **理由**：太长会超 API 限制，太短约束力不够

### 2. AI 身份约束策略
- **决策**：用"绝对禁令" + "系统终止"威慑
- **理由**：AI 模型容易出戏，必须用最强硬语言

### 3. 自然度提升策略
- **决策**：人设细节 + 语言风格 + 情感起伏
- **理由**：单纯说"自然一点"没用，要给具体指导

### 4. BGM 逻辑策略
- **决策**：没有 BGM 配置时完全隐藏控制栏
- **理由**：避免用户困惑

## 四、踩坑记录

### 坑1：请求体过大导致 API 400
- **现象**：AI 不回复，控制台报 400 错误
- **原因**：System Prompt 太长
- **解决**：增加 body-parser 限制到 100mb + 精简 prompt

### 坑2：localStorage 缓存导致 BGM 混乱
- **现象**：没设置 BGM，但还是播放了
- **原因**：之前的配置还在 localStorage 里
- **解决**：清理 localStorage + 增加检测逻辑

### 坑3：AI 多次说"我是 AI"
- **现象**：无论怎么约束，AI 还是会说
- **原因**：约束语言不够强硬
- **解决**：用"绝对禁令" + "系统终止"威慑

### 坑4：AI 违反深度定制设定
- **现象**：AI 说去过北京（设定是没出过上海）
- **原因**：背景设定约束不够明确
- **解决**：多处重复强调 + 给出具体示例
