## 1️⃣ 项目概述

### 1.1 项目简介
AI Companion 是一个基于 Web 的 AI 伴侣视频通话应用，提供智能对话、情感交互、记忆管理等功能。用户可以通过浏览器与 AI 进行自然对话，系统会记住对话历史、理解情感状态、并提供个性化的响应。

### 1.2 核心特性
- 🧠 **智能记忆系统**：自动分类、搜索、时间线展示
- 💬 **自然语言理解**：情感分析、意图识别
- ❤️ **好感度系统**：动态追踪用户关系
- 😊 **情感深度系统**：多层次情感表达
- 🤖 **AI 原则系统**：伦理合规、透明度保障
- 🎭 **人格一致性**：角色设定与行为一致
- ⏰ **防沉迷系统**：健康使用时长监控
- 📦 **记忆导出**：JSON/Markdown 格式导出
- 💓 **虚拟生理状态**：模拟能量、情绪波动
- 🎵 **高级语音节奏**：动态语速、停顿控制
- 🗣️ **SSML 情感增强**：语音情感表达
- 🔄 **重叠说话系统**：自然对话打断与恢复

### 1.3 技术亮点
- ✅ 原生 JavaScript（无框架依赖）
- ✅ IIFE 模块化设计
- ✅ LocalStorage 数据持久化
- ✅ Canvas 数据可视化
- ✅ 实时性能监控
- ✅ 完整的错误处理
- ✅ 移动端适配

## 5️⃣ 快速启动

### 5.1 环境要求
- **Node.js**: v18.0.0 或更高
- **npm**: v8.0.0 或更高
- **浏览器**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **操作系统**: macOS, Windows, Linux

### 5.2 安装步骤

#### 步骤 1：克隆仓库
```bash
git clone https://github.com/Everett-Leon/Amadeus.git
cd Amadeus/ai-companion
```

#### 步骤 2：安装依赖
```bash
npm install
```

#### 步骤 3：配置环境变量（可选）
创建 `.env` 文件（如需要）：
```env
PORT=3000
GLM_API_KEY=your_api_key_here
FISH_AUDIO_KEY=your_fish_audio_key_here
```

#### 步骤 4：启动服务器
```bash
node server.js
```

或使用 macOS 快捷启动：
```bash
chmod +x start.command
./start.command
```

#### 步骤 5：访问应用
打开浏览器访问：
```
http://localhost:3000
```

### 5.3 验证安装
访问页面后，检查以下功能：
- ✅ 页面正常加载
- ✅ 控制台无错误
- ✅ 可以发送消息

## 7️⃣ API 文档

### 7.1 后端 API 端点

#### POST /api/chat
**功能**：发送消息给 AI  
**请求体**：
```json
{
  "message": "你好",
  "context": {
    "affection": 50,
    "mood": "neutral"
  }
}
```
**响应**：
```json
{
  "reply": "你好！很高兴见到你！",
  "emotion": "happy",
  "affectionChange": 2
}
```

#### GET /api/memory
**功能**：获取所有记忆  
**响应**：
```json
{
  "memories": [
    {
      "id": "mem_001",
      "content": "用户喜欢音乐",
      "timestamp": 1234567890,
      "tags": ["兴趣", "音乐"]
    }
  ]
}
```

#### POST /api/memory/export
**功能**：导出记忆数据  
**请求体**：
```json
{
  "format": "json"  // 或 "markdown"
}
```

## 🔟 已知问题与限制

### 10.1 已知问题
✅ **目前无已知 Bug**

### 10.2 技术限制
1. **LocalStorage 限制**
   - 存储容量：约 5-10MB（浏览器依赖）
   - 解决方案：定期导出数据，或升级到 IndexedDB

2. **AI 响应延迟**
   - 依赖外部 API（GLM-4）
   - 网络延迟可能影响体验
   - 解决方案：添加加载动画，优化 Prompt

3. **3D 功能未实现**
   - 当前为 2D 界面
   - 需要集成 Three.js 或 Live2D

4. **移动端体验**
   - 已适配，但可能需要进一步优化
   - Canvas 在小屏幕上显示可能需要调整

### 10.3 性能瓶颈
- **SSML 系统**：已优化（83% 性能提升）
- **Canvas 渲染**：已优化（60fps）
- **记忆搜索**：数据量大时可能变慢（建议升级到全文搜索）
