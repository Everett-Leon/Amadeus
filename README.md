# 🌸 AI Companion — AI 伙伴视频通话

> 模拟视频通话界面的 AI 伴侣应用。通话对象是可自定义的二次元 3D 角色，通过自然语言驱动交互，具备丰富的情感表达和对话记忆。

感觉有点像嘎拉给木和会说话的汤姆猫的集合体，但交互深度远超两者。

## ✨ 核心特性

### 🎭 3D 角色交互
精细建模的二次元角色模型，支持骨骼绑定与 Blend Shapes 面部表情系统。她会有跟真人类似的情感——你讲笑话她会捂着嘴笑，你夸她她会脸红，丰富的微表情让交互更真实。

### 💬 自然语言驱动
所有交互通过对话触发，无需按键或特定操作。

- "给我翻书讲个故事" → 她走开消失，捧着一本书回来，边翻边讲
- "给我唱首歌" → 她开始唱歌
- "换套衣服" → 她换了造型回来

### 🧠 自定义人设与记忆
角色的性格由你定义，她的行为逻辑严格符合人设。更准确地说，你给了她「记忆」和「人设」。

AI 具备跨会话记忆——两天后问她前天聊了什么，她能接上。记忆有阈值，聊多了旧的内容会被逐渐遗忘，就像真人一样。

### 🎬 丰富的开场过场
每次接通视频时，角色以不同的状态登场：

- 第一次：托着头，然后恢复为正常对话
- 第二次：伸了个懒腰
- 第三次：换了套衣服走过来
- ……

正常对话时的待机状态固定，但进入方式随机变化，至少 2 种以上。

### 🔊 语音合成
角色对话支持多种声线和情绪语调。效果参考百度地图语音包，但要求更好——她说话要有情绪、有温度，不像传统 TTS 那样机械。

### 🎨 素材库
面向用户的素材库，人物模型、语音包、背景场景均可自由选择和搭配。

### 📱 移动端 + AR
支持 iOS 和 Android。打开摄像头，带她看看祖国的大好河山——通过 AR 让角色与现实环境互动。

## 🛠 技术栈

| 模块 | 技术 |
|------|------|
| 3D 引擎 | Unity |
| AI 对话 | GLM / GPT |
| 语音合成 | Azure Neural TTS / Fish Audio |
| 后端 | Node.js |
| 移动端 | Unity Export（ARKit + ARCore） |

```bash

npm start
```

打开浏览器访问 [http://localhost:3000](http://localhost:3000)

## 🗺 开发路线

- [x] **Phase 0** — 2D 原型验证（情绪系统、TTS、BGM、立绘切换）
- [ ] **Phase 1** — Unity 3D 角色基础：建模、绑定、角色能动起来
- [ ] **Phase 2** — 对话 + 表情联动：AI 说话时对口型、做表情
- [ ] **Phase 3** — 复杂交互：翻书、唱歌、走动等动画序列 + NLU 触发
- [ ] **Phase 4** — 移动端 + 摄像头 AR
- [ ] **Phase 5** — 素材库 + 多角色 + 用户系统
- [ ] **记忆系统** — 分层记忆（短期 / 摘要 / 关键记忆 + 向量检索）
