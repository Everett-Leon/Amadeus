# 记忆不完美机制设计 - 让 AI 伴侣更真实

## 核心理念

**真正的人类从不完美**：
- 💭 偶尔会记错细节（时间、地点、具体内容）
- 😳 意识到错误后会害羞地纠正自己
- 🤔 权重低的记忆更容易记错
- 💖 重要的事情不会记错（但可能记混细节）

---

## 一、记忆模糊度系统

### 1.1 模糊度计算

```javascript
// 模糊度 = 100 - 权重
fuzzyLevel = 100 - memory.weight

fuzzyLevel >= 70  → 很容易记错（权重 ≤ 30）
fuzzyLevel >= 50  → 可能记错（权重 ≤ 50）
fuzzyLevel >= 30  → 偶尔记错（权重 ≤ 70）
fuzzyLevel < 30   → 很少记错（权重 > 70）
```

### 1.2 记错概率

```javascript
// 根据模糊度计算记错概率
mistakeProbability = {
  fuzzyLevel >= 70: 40%,  // 很容易记错
  fuzzyLevel >= 50: 20%,  // 可能记错
  fuzzyLevel >= 30: 10%,  // 偶尔记错
  fuzzyLevel < 30:  2%,   // 很少记错
  isPermanent: 0%,        // 永久记忆不会记错核心内容
}
```

## 三、纠正机制

### 3.1 纠正触发条件

1. **用户直接纠正**："不是X，是Y"
2. **用户否定**："不对"、"不是"、"错了"
3. **用户重复强调**：用户再次提到正确信息

### 3.2 纠正反应模式

根据性格不同，纠正反应也不同：

#### 傲娇型（三鹰朝）
```
AI: "嗯...你喜欢吃蓝莓对吧？[thinking]"
用户: "是草莓啦"
AI: "哼...我、我当然知道是草莓啦 [embarrassed]...只是刚才一时口误而已 [shy]...才不是记错了呢 [angry]"
```

#### 害羞型
```
AI: "嗯...你喜欢吃蓝莓对吧？[thinking]"
用户: "是草莓啦"
AI: "诶...对不起对不起 [embarrassed]...我记错了...好丢人啊 [shy]...是草莓呢 [sad]"
```

#### 活泼型
```
AI: "嗯...你喜欢吃蓝莓对吧？[thinking]"
用户: "是草莓啦"
AI: "啊哈哈...我记错啦 [embarrassed]...是草莓是草莓！[happy] 抱歉抱歉～[shy]"
```

### 3.3 纠正后的记忆强化

```javascript
// 用户纠正后，记忆会被强化
if (用户纠正) {
  memory.weight += 25;  // 大幅强化
  memory.accessCount++;
  memory.lastAccessed = now;
  
  // 标记为"被纠正过"，下次不会再犯同样的错误
  memory.correctedCount = (memory.correctedCount || 0) + 1;
  
  // 如果被纠正2次以上，升级重要性
  if (memory.correctedCount >= 2) {
    memory.importance = 'high';
  }
}
```

## 五、实现细节

### 5.1 记错触发逻辑

```javascript
// 在 AI 回复时，根据记忆权重决定是否记错
function shouldMakeMistake(memory) {
  // 永久记忆的核心内容不会记错
  if (memory.isPermanent && memory.type === 'event') {
    return false;
  }
  
  // 计算模糊度和记错概率
  const fuzzyLevel = 100 - memory.weight;
  let probability = 0;
  
  if (fuzzyLevel >= 70) probability = 0.4;
  else if (fuzzyLevel >= 50) probability = 0.2;
  else if (fuzzyLevel >= 30) probability = 0.1;
  else probability = 0.02;
  
  // 如果之前被纠正过，降低记错概率
  if (memory.correctedCount > 0) {
    probability *= 0.5;
  }
  
  return Math.random() < probability;
}

// 生成记错的内容
function generateMistake(memory) {
  const mistakeTypes = ['time', 'detail', 'confusion'];
  const type = mistakeTypes[Math.floor(Math.random() * mistakeTypes.length)];
  
  switch (type) {
    case 'time':
      return generateTimeMistake(memory);
    case 'detail':
      return generateDetailMistake(memory);
    case 'confusion':
      return generateConfusionMistake(memory);
  }
}
```

### 5.2 纠正检测逻辑

```javascript
// 检测用户是否在纠正 AI
function detectCorrection(userMessage, aiLastMessage) {
  const correctionKeywords = [
    '不是', '不对', '错了', '是...啦', '是...呀',
    '不是...是...', '应该是', '其实是'
  ];
  
  return correctionKeywords.some(k => userMessage.includes(k));
}

// 处理纠正
function handleCorrection(memory, correctInfo) {
  // 更新记忆内容
  memory.content = correctInfo;
  
  // 大幅强化权重
  memory.weight = Math.min(100, memory.weight + 25);
  
  // 记录纠正次数
  memory.correctedCount = (memory.correctedCount || 0) + 1;
  
  // 更新访问信息
  memory.accessCount++;
  memory.lastAccessed = new Date().toISOString();
  
  // 如果被纠正2次以上，升级重要性
  if (memory.correctedCount >= 2 && memory.importance !== 'critical') {
    memory.importance = 'high';
  }
  
  return memory;
}
```

## 六、记错频率控制

### 6.1 全局记错计数器

```javascript
// 避免记错太频繁，影响用户体验
const mistakeTracker = {
  lastMistakeTime: null,
  mistakeCount: 0,
  maxMistakesPerSession: 3,  // 每次通话最多记错3次
  minIntervalMinutes: 5,      // 两次记错至少间隔5分钟
};

function canMakeMistake() {
  const now = Date.now();
  
  // 检查是否超过最大次数
  if (mistakeTracker.mistakeCount >= mistakeTracker.maxMistakesPerSession) {
    return false;
  }
  
  // 检查时间间隔
  if (mistakeTracker.lastMistakeTime) {
    const elapsed = (now - mistakeTracker.lastMistakeTime) / 1000 / 60;
    if (elapsed < mistakeTracker.minIntervalMinutes) {
      return false;
    }
  }
  
  return true;
}

function recordMistake() {
  mistakeTracker.lastMistakeTime = Date.now();
  mistakeTracker.mistakeCount++;
}
```

### 6.2 记错频率建议

| 通话时长 | 建议记错次数 | 用户感受 |
|---------|------------|---------|
| 5分钟 | 0-1次 | 自然 |
| 15分钟 | 1-2次 | 真实 |
| 30分钟 | 2-3次 | 可爱 |
| 60分钟+ | 3-5次 | 不会太多 |
