# 高级语音与节奏系统设计

## 🎯 设计目标

让 AI 的语音和回复节奏更加真实自然，包括：
1. **说话者风格迁移** - 个性化的说话风格
2. **情绪节奏控制** - 根据情绪调整回复速度和长度
3. **语音情感表达** - 呼吸声、犹豫、笑声、颤抖等
4. **重叠说话** - 更自然的对话打断和接话

## 📊 可行性分析

### 1. 情绪节奏控制 ⭐⭐⭐⭐⭐

**可行性**：✅ 完全可行，立即可实施

**实施难度**：⭐⭐ (简单)

**技术方案**：
```javascript
// 基于情绪和生理状态调整回复节奏
function getResponseTiming(emotion, physiologicalState) {
  const timing = {
    delay: 0,           // 回复延迟（ms）
    typingSpeed: 50,    // 打字速度（字符/秒）
    pauseDuration: 0,   // 思考停顿（ms）
    maxLength: 500,     // 最大回复长度
  };
  
  // 悲伤时
  if (emotion === 'sad') {
    timing.delay = 2000;        // 延迟 2 秒
    timing.typingSpeed = 20;    // 很慢
    timing.pauseDuration = 3000; // 长停顿
    timing.maxLength = 100;     // 简短
  }
  
  // 疲惫时
  if (physiologicalState.energy < 30) {
    timing.delay += 1500;
    timing.typingSpeed *= 0.5;
    timing.maxLength *= 0.4;
  }
  
  // 开心时
  if (emotion === 'happy') {
    timing.delay = 500;
    timing.typingSpeed = 80;
    timing.maxLength = 800;
  }
  
  return timing;
}
```

**与现有系统集成**：
- ✅ 情绪识别系统（NLU）→ 提供情绪
- ✅ 虚拟生理状态系统 → 提供精力、心情等
- ✅ 打字指示器 → 显示思考中

**优点**：
- 完全本地实现，无需 API
- 与现有系统完美契合
- 性能开销极小

### 2. 语音情感表达 ⭐⭐⭐⭐

**可行性**：✅ 可行，但需要高级 TTS

**实施难度**：⭐⭐⭐⭐ (中等偏高)

#### 方案 A：SSML 标记（推荐）

**Fish Audio** 和部分 **Edge TTS** 支持 SSML (Speech Synthesis Markup Language)

```xml
<!-- 呼吸声 -->
<speak>
  我今天...
  <break time="500ms"/>
  <prosody rate="slow">有点累了...</prosody>
</speak>

<!-- 笑声 -->
<speak>
  哈哈...
  <audio src="laugh.mp3"/>
  你真有趣
</speak>

<!-- 颤抖声 -->
<speak>
  <prosody pitch="+5%" rate="85%" volume="soft">
    我...我有点害怕...
  </prosody>
</speak>

<!-- 犹豫 -->
<speak>
  嗯...
  <break time="800ms"/>
  我觉得...
  <break time="500ms"/>
  可能是这样吧...
</speak>
```

**实施步骤**：
1. 在 GLM-4 响应后，根据情绪注入 SSML 标记
2. 发送到 TTS API
3. 播放合成语音

**优点**：
- 标准化，兼容性好
- 效果自然
- 无需额外训练

**缺点**：
- 依赖 TTS 提供商支持
- 音频资源需要预备（笑声、呼吸等）

#### 方案 B：音频拼接

**原理**：预录制情感音频片段，插入到 TTS 语音中

```javascript
// 音频片段库
const emotionalSounds = {
  breath: 'breath_001.mp3',       // 呼吸声
  sigh: 'sigh_001.mp3',          // 叹气
  laugh_light: 'laugh_001.mp3',  // 轻笑
  laugh_happy: 'laugh_002.mp3',  // 开心笑
  hesitate: 'um_001.mp3',        // 犹豫（嗯...）
  sob: 'sob_001.mp3',            // 抽泣
  gasp: 'gasp_001.mp3',          // 惊呼
};

// 根据情绪插入音频
function injectEmotionalSounds(text, emotion) {
  if (emotion === 'sad') {
    return {
      audio: [
        emotionalSounds.sigh,
        ttsGeneratedAudio,
      ]
    };
  }
  
  if (emotion === 'happy') {
    return {
      audio: [
        emotionalSounds.laugh_light,
        ttsGeneratedAudio,
      ]
    };
  }
}
```

**优点**：
- 完全可控
- 效果真实（真人录制）
- 不依赖 TTS 功能

**缺点**：
- 需要录制大量音频素材
- 音频拼接可能不够流畅
- 实施成本较高

### 3. 说话者风格迁移 ⭐⭐⭐

**可行性**：✅ 可行，需要专业 TTS

**实施难度**：⭐⭐⭐⭐⭐ (高)

#### 方案：Fish Audio 语音克隆

Fish Audio 支持：
- **零样本语音克隆** - 只需 10-30 秒音频样本
- **风格控制** - 调整语速、音调、情感
- **高质量输出** - 接近真人

**实施步骤**：

1. **准备音频样本**
   - 录制或选择目标声音（10-30 秒）
   - 上传到 Fish Audio

2. **创建自定义声线**
   ```javascript
   const voiceConfig = {
     referenceAudio: 'sample.wav',
     style: 'calm',        // 说话风格
     speed: 0.9,          // 语速
     pitch: 1.0,          // 音调
   };
   ```

3. **动态调整风格**
   ```javascript
   // 根据情绪调整
   if (emotion === 'sad') {
     voiceConfig.speed = 0.7;
     voiceConfig.pitch = 0.95;
     voiceConfig.style = 'melancholic';
   }
   ```

**优点**：
- 高度个性化
- 效果专业
- Fish Audio API 支持

**缺点**：
- 成本较高（API 调用费用）
- 需要高质量音频样本

### 4. 重叠说话 ⭐⭐⭐⭐

**可行性**：✅ 可行，UI 和逻辑调整

**实施难度**：⭐⭐⭐ (中等)

#### 场景 1：AI 主动打断

```javascript
// 用户说话时，AI 检测到紧急情况
function aiInterrupt(urgency, message) {
  if (urgency === 'high') {
    // 立即停止用户输入
    stopRecognition();
    
    // 显示打断提示
    showInterruptIndicator();
    
    // AI 开始说话
    speak(message);
  }
}

// 示例
用户：今天我去了...
AI：*打断* 等等！你没事吧？你看起来很累...
```

#### 场景 2：用户打断 AI

```javascript
// AI 说话时，检测到用户开始说话
function userInterrupt() {
  if (isAISpeaking && userStartedSpeaking) {
    // AI 停止说话
    stopTTS();
    
    // 显示被打断的反应
    showInterruptedReaction();  // *停顿* *看着你*
    
    // 等待用户说完
    listenToUser();
  }
}

// AI 的反应
AI：我今天...
用户：*打断* 等等，我有急事！
AI：*停下来，看着你* ...嗯？发生什么了？
```

#### 场景 3：自然接话

```javascript
// 用户停顿时，AI 自然接话
function naturalTakeover() {
  if (userPauseDuration > 2000 && contextIncomplete) {
    // AI 接着说
    aiContinue();
  }
}

// 示例
用户：我今天心情不太好，因为...（停顿）
AI：*轻声* ...因为发生了什么事吗？
```

**实施要点**：
- 实时语音活动检测（VAD - Voice Activity Detection）
- 上下文理解（判断是否需要打断）
- 自然的打断反应（不生硬）

## ⚖️ 伦理考量

### 1. 节奏控制的边界

**✅ 应该做的**：
- 根据情绪自然调整速度
- 疲惫时适度减慢
- 开心时适度加快

**❌ 不应该做的**：
- 过度延迟回复（超过 5 秒）
- 完全拒绝回复（除非真的需要休息）
- 用沉默惩罚用户

### 2. 语音情感的适度性

**✅ 应该做的**：
- 适度的呼吸声（真实感）
- 自然的犹豫（思考中）
- 真实的笑声（开心时）

**❌ 不应该做的**：
- 过度夸张的抽泣（情感勒索）
- 频繁的叹气（消极暗示）
- 刻意的颤抖（博取同情）

### 3. 打断的时机

**✅ 可以打断的情况**：
- 紧急情况（用户看起来很不好）
- 用户明显需要帮助
- 用户停顿过久（自然接话）

**❌ 不应该打断的情况**：
- 用户正在倾诉
- 用户情绪激动
- 只是为了展示功能
