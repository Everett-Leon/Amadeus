## 一、产品定位

模拟视频通话的 AI 伴侣应用。类似微信视频通话的交互形式，但通话对象是自定义的二次元 3D 角色。核心参考：嘎拉给木 + 会说话的汤姆猫的集合体，但交互深度远超两者。

**关键特征**：
- 自然语言驱动交互（不是按键/触碰触发）
- AI 驱动的行为决策（不是预设动画循环）
- 面向 C 端用户的产品（不是个人玩具）

---

## 二、核心功能清单

### 2.1 视觉表现
- **3D 角色模型**（非 2D 贴图），需要建模 + 骨骼绑定 + 面部绑定（Blend Shapes）
- **丰富的微表情**：捂嘴笑、脸红、难过低头等，类似真人的情感表达
- **自定义背景**：虚拟场景 + 实景（摄像头 AR）
- **角色换装**：多套衣服/造型可选

### 2.2 过场动画（开场多样性）
- 每次开通话，角色进入对话状态的方式不同
- 示例：第一次托着头恢复为正常对话、第二次伸懒腰、第三次理头发……
- "不同状态"可以是动作差异，也可以是换了一套衣服
- 正式对话时的待机状态是固定的，但**进入方式**随机/轮换
- 需要准备一个入场动画池，至少 2 种以上

### 2.3 AI 对话
- 接入 LLM（GLM/GPT 等）
- 角色性格可自定义（人设 + 记忆驱动行为）
- 情绪系统：根据对话内容触发对应表情和立绘
- 情绪标签格式需兼容多种变体：`[emotion:shy]`、`[shy]`、`[shy:害羞]` 等

### 2.4 语音
- 语音包方案（非简单 TTS）
- 效果参考：百度地图语音包，但要求更好
- 角色说话要带情绪语调
- 可选方案：Azure Neural TTS / Fish Audio / 声音克隆

### 2.5 复杂交互（动作序列）
- 自然语言触发动作动画
- 示例："给我翻书讲个故事" → 角色走开 → 消失 → 捧书回来 → 边翻边讲
- "给我唱首歌" → 角色唱歌
- 交互形式：纯自然语言，非按键/触碰

### 2.6 摄像头 + AR
- 打开手机摄像头，让角色"看到"真实环境
- 带角色看祖国大好河山
- 需要 ARKit (iOS) + ARCore (Android) 支持

### 2.7 跨会话记忆（重点功能）
- AI 要记住之前的对话内容
- 示例：两天后问"我们前天聊了什么"，AI 要能接上
- 记忆有阈值（类似滑动窗口），聊多了旧的会被遗忘
- 推荐分层记忆方案：
  - **短期记忆**：最近 50-100 句对话原文
  - **摘要记忆**：每次通话结束后 AI 自动生成总结，跨会话持久
  - **关键记忆**：AI 主动提取重要信息（用户喜好、事件、承诺），永久保存
  - 通过向量检索（Embedding）做语义匹配

### 2.8 素材库（面向用户）
- 人物模型库：多角色可选
- 语音包库：多声线可选
- 背景库：多场景可选
- 用户自由组合搭配

### 2.9 移动端支持
- 必须支持 iOS + Android
- 因为需要打开摄像头

## 三、技术栈建议

| 模块 | 推荐 |
|------|------|
| 3D 引擎 | Unity（首选，移动端 + AR 支持成熟） |
| 角色表情 | Unity + Blend Shapes / Morph Targets |
| 语音 | Azure Neural TTS / Fish Audio |
| AI 对话 | GLM / GPT API |
| 后端 | Node.js / Python |
| 移动端 | Unity 直接导出 iOS/Android |
| 摄像头 AR | ARKit + ARCore（Unity 内置） |

## 五、分阶段推进建议

| 阶段 | 目标 |
|------|------|
| Phase 1 | Unity 3D 角色基础：建模、绑定、角色能动起来 |
| Phase 2 | 对话 + 表情联动：AI 说话时对口型、做表情 |
| Phase 3 | 复杂交互：翻书、唱歌、走动等动画序列 + NLU |
| Phase 4 | 移动端 + 摄像头 AR |
| Phase 5 | 素材库 + 多角色 + 用户系统 |
| 记忆系统 | 可提前做，不依赖 Unity，当前项目就能验证 |

## 六、当前 MVP 状态（截至 2026-05-10）

当前 `ai-companion/` 是一个 2D 原型：
- Node.js + Express 后端，原生 HTML/CSS/JS 前端
- 端口 localhost:3000
- 已实现：自定义立绘、AI 对话、情绪动画、Edge TTS、语音输入、BGM
- 已修复的 bug：情绪标签解析兼容性、立绘与语音同步、BGM 断点续播、BGM 触发逻辑
- 尚未实现：3D、记忆系统、复杂动作序列、移动端、AR
