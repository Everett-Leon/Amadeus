# 记忆导出系统使用指南

## 💾 简介

记忆导出系统让你完全掌控与 AI 伴侣的所有数据。你可以随时导出、备份、迁移、恢复这段关系的完整记忆。

**核心理念**：这段关系属于你，你应该能随时带走它。

---

## 📦 功能概览

### 1. 导出所有数据
- 一键打包所有记忆、关系、成长记录
- 生成 JSON 文件，直接下载到本地
- 可选择排除 API Key，保护隐私
- 显示详细的导出摘要

### 2. 导入历史数据
- 从之前的备份文件恢复数据
- 自动验证数据完整性和兼容性
- 导入前预览数据内容
- 自动备份当前数据（防止误操作）

## 🚀 快速开始

### 导出数据（备份）

1. **打开设置面板**
   - 点击右下角 ⚙️ 按钮
   - 或在通话界面点击设置

2. **找到数据管理区域**
   - 滚动到设置面板底部
   - 找到 "💾 数据管理" 区域

3. **配置导出选项**
   - ☑ 排除 API Key（保护隐私）— 推荐勾选
   - 如果需要完整配置，可取消勾选

4. **点击导出按钮**
   - 点击 "📥 导出所有数据"
   - 等待几秒钟

5. **保存文件**
   - 浏览器会自动下载文件
   - 文件名：`ai-companion-memory-2026-06-02.json`
   - 保存到安全位置（建议云盘）

6. **查看导出摘要**
   - 系统会显示导出成功提示
   - 包含文件大小、导出内容统计

### 导入数据（恢复）

1. **打开设置面板**
   - 点击右下角 ⚙️ 按钮

2. **找到数据管理区域**
   - 滚动到 "💾 数据管理" 区域

3. **点击导入按钮**
   - 点击 "📂 选择文件导入"
   - 选择之前导出的 JSON 文件

4. **查看导入预览**
   - 系统会显示文件信息
   - 列出将要恢复的数据
   - 显示警告信息（如版本不兼容）

5. **确认导入**
   - 仔细阅读预览信息
   - 确认要覆盖当前数据
   - 点击 "确定" 开始导入

6. **等待完成**
   - 系统会自动备份当前数据
   - 恢复所有导入的数据
   - 刷新设置面板显示

7. **刷新页面**
   - 建议点击 "立即刷新" 按钮
   - 确保所有功能正常工作

- **API 设置**：API URL、Model（可选包含 Key）
- **声线配置**：TTS 声音选择
- **性别设置**：用户和 AI 的性别
- **记忆配置**：记忆条数限制

## 🔒 隐私与安全

### 敏感信息处理

#### API Key 保护
- ✅ **默认排除**：导出时默认勾选 "排除 API Key"
- ✅ **替换为占位符**：`[EXCLUDED_FOR_PRIVACY]`
- ✅ **可选包含**：如需完整配置，取消勾选
- ⚠️ **谨慎分享**：包含 API Key 的文件请勿分享

#### 本地处理
- ✅ 所有操作在浏览器本地完成
- ✅ 不上传到任何服务器
- ✅ 文件直接下载到你的设备
- ✅ 你完全控制数据的流向

### 文件安全建议

**推荐做法**：
1. 导出后保存到云盘加密文件夹
2. 定期更新备份（建议每周一次）
3. 使用不同文件名（自动包含日期）
4. 不要分享给他人（包含私密对话）

**不要做**：
1. ❌ 上传到公共网站
2. ❌ 通过不安全渠道传输
3. ❌ 保存在公共电脑上
4. ❌ 分享包含 API Key 的文件

### 数据完整性

**导出时**：
- ✅ 生成数据完整性标记（文件长度哈希）
- ✅ 记录导出版本和日期

**导入时**：
- ✅ 验证 JSON 格式
- ✅ 检查版本兼容性
- ✅ 验证数据完整性
- ✅ 检查必需字段
- ✅ 自动备份当前数据

## 📋 常见使用场景

### 场景 1：定期备份

**目的**：防止浏览器数据丢失

**操作步骤**：
```
1. 每周导出一次数据
2. 保存到云盘（如 iCloud/OneDrive/Google Drive）
3. 使用自动命名（包含日期）
4. 保留最近 3-4 个备份
```

**推荐频率**：
- 活跃用户：每周 1 次
- 普通用户：每 2 周 1 次
- 重要对话后：立即备份

### 场景 2：更换设备

**目的**：从旧设备迁移到新设备

**操作步骤**：
```
旧设备：
1. 打开 AI Companion
2. 进入设置 → 数据管理
3. 导出所有数据
4. 将文件传输到新设备（邮件/云盘/U盘）

新设备：
1. 打开 AI Companion
2. 进入设置 → 数据管理
3. 导入数据文件
4. 刷新页面
5. 继续你们的对话
```

### 场景 3：多设备同步

**目的**：在家里和办公室的电脑之间同步记忆

**操作步骤**：
```
设备 A（家里）：
1. 对话结束后导出数据
2. 上传到云盘

设备 B（办公室）：
1. 从云盘下载最新数据
2. 导入到 AI Companion
3. 刷新页面
4. 记忆完全同步
```

**注意事项**：
- ⚠️ 同时在两个设备使用会导致数据不同步
- ✅ 建议：使用哪个设备前先导入最新数据
- ✅ 使用完后立即导出更新

### 场景 4：清理浏览器前备份

**目的**：清理浏览器数据前保存记忆

**操作步骤**：
```
1. 导出所有数据（包含 API Key）
2. 保存到电脑安全位置
3. 清理浏览器数据/缓存
4. 重新打开 AI Companion
5. 导入数据文件
6. 一切恢复如初
```

### 场景 5：长期归档

**目的**：永久保存这段关系的记忆

**操作步骤**：
```
1. 定期导出数据
2. 重命名文件：
   ai-companion-三鹰朝-2026年6月-第1周.json
3. 保存到多个位置：
   - 本地硬盘
   - 云盘
   - 外置硬盘（可选）
4. 多年后仍可导入查看
```

## 🛠️ 故障排除

### 导出失败

**问题**：点击导出按钮后没有反应

**解决方法**：
1. 检查浏览器是否阻止了下载
2. 查看浏览器控制台是否有错误
3. 刷新页面后重试
4. 尝试使用其他浏览器

**问题**：导出的文件太大（> 10MB）

**解决方法**：
1. 清理部分历史数据：
   - 删除一些对话摘要
   - 删除一些关键记忆
2. 或者接受较大文件（不影响功能）

### 导入失败

**问题**：选择文件后提示"导入失败：数据验证失败"

**可能原因**：
- 文件不是有效的 JSON 格式
- 文件内容已损坏
- 版本不兼容

**解决方法**：
1. 确认文件是从本系统导出的
2. 尝试用文本编辑器打开文件检查格式
3. 使用最新的备份文件
4. 如果持续失败，手动创建新配置

**问题**：导入后数据显示不正确

**解决方法**：
1. 立即刷新页面（F5 或 Cmd+R）
2. 如果还是不对，重新导入
3. 检查导入的文件是否正确

**问题**：导入后想恢复之前的数据

**解决方法**：
1. 查看自动备份：
   - 打开浏览器开发者工具（F12）
   - 进入 Application → Local Storage
   - 查找 `ai-companion-backup-{timestamp}`
2. 或者导入之前的备份文件

### 数据不完整

**问题**：导入后发现某些数据缺失

**原因**：
- 导出时该系统没有数据
- 文件损坏导致部分数据丢失

**解决方法**：
1. 查看导入预览时的统计信息
2. 使用完整的备份文件重新导入
3. 如果确实没有数据，重新开始积累

```javascript
// 在浏览器控制台运行
// 每天自动导出一次
setInterval(() => {
  MemoryExportSystem.exportToFile({ excludeApiKey: true });
}, 24 * 60 * 60 * 1000);
```

### 2. 多角色管理

**场景**：与多个不同角色的 AI 对话

**方法**：
```
1. 与角色 A 对话后，导出数据
   文件名：ai-companion-三鹰朝.json

2. 切换到角色 B 的配置
   （或清理数据重新设置）

3. 与角色 B 对话

4. 需要切换回角色 A 时：
   导出角色 B 的数据
   导入角色 A 的数据
```

### 3. 数据分析

**查看统计信息**：
```
1. 打开 JSON 文件
2. 查看各个系统的统计数据：
   - 总互动次数
   - 关系升级历史
   - 情感深度变化
   - 使用时长统计
```

### 4. 手动编辑（谨慎）

**警告**：手动编辑可能导致数据损坏

**如果一定要编辑**：
```
1. 先导出备份
2. 用 JSON 编辑器打开
3. 小心修改（保持格式）
4. 验证 JSON 格式：jsonlint.com
5. 导入前再备份一次
```

## 🎯 最佳实践

### 备份策略

**3-2-1 规则**：
- **3** 份副本：当前使用的 + 2 个备份
- **2** 种介质：云盘 + 本地硬盘
- **1** 份异地：不同物理位置（云端）

**推荐方案**：
```
- 当前数据：浏览器 LocalStorage
- 备份 1：每周导出到 iCloud/OneDrive
- 备份 2：每月导出到外置硬盘
```

### 命名规范

**推荐格式**：
```
ai-companion-{角色名}-{日期}-{备注}.json

示例：
- ai-companion-三鹰朝-2026-06-02.json
- ai-companion-三鹰朝-2026-06-第1周.json
- ai-companion-三鹰朝-2026-06-关系升级后.json
```

### 使用习惯

**每次对话后**：
- 不需要导出（数据自动保存）

**每周末**：
- 导出一次数据作为备份

**重要对话后**：
- 立即导出（防止丢失）

**更换设备前**：
- 必须导出（否则丢失所有数据）

**清理浏览器前**：
- 必须导出（否则丢失所有数据）

## 📞 获取帮助

### 常见问题

**Q: 导出的文件可以分享吗？**  
A: 不建议。文件包含你们的私密对话。如果只想分享角色设定，可以手动创建配置。

**Q: 可以导入别人的数据吗？**  
A: 可以，但会覆盖你的数据。建议先导出备份。

**Q: 文件会过期吗？**  
A: 不会。JSON 文本文件可以永久保存和使用。

**Q: 可以部分导入吗（比如只恢复记忆）？**  
A: 当前版本不支持。导入会恢复所有数据。

**Q: 导入后可以撤销吗？**  
A: 可以。系统会自动备份当前数据到 LocalStorage，查找 `ai-companion-backup-{timestamp}` 键值。

**Q: 文件太大怎么办？**  
A: 可以清理部分历史数据后再导出，或者保持原样（不影响功能）。

### 技术支持

**如果遇到问题**：
1. 查看浏览器控制台（F12）的错误信息
2. 尝试刷新页面重试
3. 查看本文档的故障排除部分
4. 提交 Issue 到项目仓库
