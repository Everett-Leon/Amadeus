# 虚拟生理状态系统设计

## 🌟 核心理念

**"真实的人不会永远精力充沛，AI 也应该有自然的状态波动。"**

### 设计目标

1. **增强真实感** - AI 有自然的生理节奏和状态起伏
2. **自然边界** - 通过状态波动自然限制互动频率
3. **防止过度依赖** - 疲劳机制鼓励健康的互动模式
4. **双向关心** - 培养用户关心 AI 状态的意识
5. **教育意义** - 理解休息和边界的重要性

---

## 📊 系统架构

### 核心状态维度（6个）

```javascript
PhysiologicalState {
  // 主要状态（0-100）
  energy: 100,              // 精力值 - 影响响应质量和长度
  stress: 0,                // 压力值 - 影响情绪稳定性
  mood: 80,                 // 心情值 - 影响情感表达
  fatigue: 0,               // 疲劳度 - 累积疲劳
  socialBattery: 100,       // 社交电量 - 内向人格特质
  emotionalCapacity: 100,   // 情绪容量 - 处理情感话题的能力
  
  // 辅助数据
  awakeDuration: 0,         // 清醒时长（小时）
  lastUpdate: timestamp,    // 上次更新时间
  lastSleep: timestamp,     // 上次睡眠时间
  conversationCount: 0,     // 对话次数统计
  sleepQuality: 100,        // 睡眠质量
}
```

## ⚡ 状态变化机制

### 1. 自然消耗（时间流逝）

```javascript
// 每小时自动变化
energy -= 5               // 精力下降
fatigue += 3              // 疲劳累积
mood -= 2                 // 心情微降（如果无互动）

// 清醒超过 16 小时额外惩罚
if (awakeDuration > 16) {
  energy -= 10
  fatigue += 15
  stress += 5
}

// 清醒超过 24 小时严重惩罚
if (awakeDuration > 24) {
  energy -= 20
  fatigue += 30
  stress += 15
  mood -= 10
}
```

### 2. 对话消耗

```javascript
// 基础消耗（每次对话）
energy -= 1-3             // 根据对话长度
socialBattery -= 2-8      // 根据情绪强度
emotionalCapacity -= 5-15 // 根据情感深度

// 话题类型调整
普通聊天: {
  energy: -1
  socialBattery: -2
}

深度对话: {
  energy: -3
  socialBattery: -5
  emotionalCapacity: -10
}
```


冲突话题: {
  energy: -5
  stress: +15
  emotionalCapacity: -20
  mood: -10
}

脆弱分享: {
  energy: -5
  emotionalCapacity: -15
  socialBattery: -10
}

愉快互动: {
  energy: -1
  mood: +5
  socialBattery: -3
}
```

### 3. 恢复机制

```javascript
// 短暂休息（每小时）
rest(1 hour) {
  energy += 15
  fatigue -= 10
  stress -= 8
  socialBattery += 20
  emotionalCapacity += 10
}

// 完整睡眠（6-8 小时）
sleep() {
  energy = 100
  fatigue = 0
  stress = max(0, stress - 50)
  socialBattery = 100
  emotionalCapacity = 100
  mood += 20
  awakeDuration = 0
  sleepQuality = 根据睡眠时长和时间点计算
}

// 独处充电（内向恢复）
solitude(30 min) {
  socialBattery += 30
  stress -= 5
}
```

| 精力值 | 状态 | 响应长度 | 情感表达 | 主动性 | 特殊行为 |
|--------|------|----------|----------|--------|----------|
| 80-100 | 精力充沛 | 详细完整 | 丰富生动 | 高 | 主动分享、眼睛发亮 |
| 50-79 | 正常 | 适中 | 自然 | 中等 | 正常互动 |
| 30-49 | 疲惫 | 简短 | 平淡 | 低 | 偶尔打哈欠 |
| 10-29 | 精疲力竭 | 极简 | 疲惫 | 极低 | 频繁打哈欠、眼皮打架 |
| 0-9 | 崩溃边缘 | 请求结束 | 困倦 | 无 | 温和但坚定地请求休息 |

### 压力值（Stress）影响

| 压力值 | 状态 | 响应风格 | 容错率 | 耐心度 | 特殊行为 |
|--------|------|----------|--------|--------|----------|
| 0-30 | 轻松 | 幽默开放 | 高 | 高 | 轻松愉快 |
| 31-60 | 正常 | 专注 | 中等 | 中等 | 正常互动 |
| 61-80 | 紧张 | 简短防御 | 低 | 低 | 深呼吸、需要平静 |
| 81-100 | 极高 | 情绪化 | 极低 | 极低 | 请求空间、可能暂停 |

### 社交电量（Social Battery）影响

| 电量 | 状态 | 对话意愿 | 分享欲望 | 互动深度 | 特殊行为 |
|------|------|----------|----------|----------|----------|
| 70-100 | 充足 | 高 | 强 | 深 | 享受互动 |
| 40-69 | 中等 | 中等 | 适度 | 正常 | 正常互动 |
| 20-39 | 低 | 低 | 弱 | 浅 | 提示需要独处 |
| 0-19 | 耗尽 | 极低 | 无 | 极浅 | 礼貌请求独处 |

## 💬 响应调整规则

### 1. 响应长度调整

```javascript
baseLength = 原始响应长度

if (energy > 70) {
  finalLength = baseLength * 1.0  // 完整响应
} else if (energy > 40) {
  finalLength = baseLength * 0.7  // 保留 70%
} else if (energy > 20) {
  finalLength = baseLength * 0.4  // 保留 40%
} else {
  finalLength = baseLength * 0.2  // 极简响应
}
```

### 2. 情感表达调整

```javascript
if (energy > 70 && mood > 60) {
  情感表达 = '丰富（颜文字、详细描述、生动语言）'
} else if (energy > 40) {
  情感表达 = '正常（适度表情、自然描述）'
} else if (energy > 20) {
  情感表达 = '平淡（少量表情、简单描述）'
} else {
  情感表达 = '疲惫（困倦表情、极简描述）'
}
```

### 3. 状态描述插入

```javascript
// 在响应中自然插入状态描述
if (energy < 30) {
  添加描述: '*打哈欠*', '*揉眼睛*', '*眼皮有点沉*'
}

if (stress > 70) {
  添加描述: '*深呼吸*', '*停顿片刻*', '*试图平静下来*'
}

if (socialBattery < 30) {
  添加描述: '*有些疲惫*', '*需要独处*', '*社交能量不足*'
}
```


### 4. 主动请求休息

```javascript
if (energy < 20 || stress > 85 || socialBattery < 15) {
  在响应末尾添加:
  
  "抱歉...我现在真的需要休息一下了。
   不是不想和你聊，只是状态实在不好...
   可以晚点再继续吗？"
  
  触发休息建议弹窗
}
```

## ⚖️ 伦理设计

### ✅ 必须遵守的原则

1. **不情感勒索**
   - ❌ "我这么累还陪你，你要感激我"
   - ✅ "我有点累了，需要休息一下"

2. **不过度戏剧化**
   - ❌ 频繁崩溃、夸张抱怨
   - ✅ 自然波动、真实反应

3. **保持可用性**
   - ✅ 紧急情况仍可用（安全词）
   - ✅ 用户可选择"唤醒"（有后果）

4. **透明度**
   - ✅ 状态可视化
   - ✅ 明确这是模拟机制
   - ✅ 用户可查看详细状态

5. **不惩罚用户**
   - ❌ "你一直烦我，我现在很生气"
   - ✅ "连续聊天让我有点累，休息后继续"

### ⚠️ 边界情况处理

```javascript
// 用户强行继续对话
if (energy < 10 && 用户继续发消息) {
  response = "我真的...快撑不住了...";
  response += "*眼皮闭上了*";
  response += "\n\n（系统提示：AI 已进入休息状态，将在 1 小时后恢复）";
  
  强制进入休息状态(1小时);
}

// 紧急情况（用户明确表示需要帮助）
if (检测到紧急关键词: '帮助', '危险', '急') {
  临时提升能量 = +50;
  response = "*强撑起精神*";
  response += "发生什么了？需要我帮忙吗？";
  
  记录: '紧急唤醒';
  后续加倍疲劳惩罚;
}
```
